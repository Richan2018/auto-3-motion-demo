总览规则

核心原则：先记录体验者想把哪些图标加入列表，再根据 agentbar 左侧剩余空间决定哪些图标直接显示、哪些图标先收进紫色 more。agentbar 变宽时，图标列表会主动让出空间；点击紫色 more 展开时，则临时把已加入的图标全部摊开展示。

模拟图标点击

先记录用户想加入或移除哪个图标，再根据 agentbar 左侧剩余空间决定哪些图标显示、哪些图标收进 more。

普通自动折叠

如果图标列表快要碰到 agentbar，就从尾部开始收起图标，并把紫色 more 放在最后一个安全的位置。

紫色 more 展开

点击紫色 more 后，把所有已加入图标摊开展示；more 向后移动多少，agentbar 就跟着向右退多少并淡出。

紫色 more 收起

再次点击 more 时，不回到旧位置，而是按当前 agentbar 宽度重新判断应该收起哪些图标。

Agentbar 宽度变化

胶囊或黄色更多让 agentbar 变宽变窄时，图标列表会同步判断是否需要让出空间或补回图标。

状态含义讲解重点

左右胶囊是否显示当前是否隐藏了左胶囊或右胶囊只剩一个胶囊时，AI 和剩余胶囊之间不留空隙

案例分类

图标列表基础

01 · 图标列表基础

单个图标隐藏与恢复

前提
初始状态，头像 + 图标1-图标6 显示，紫色 more 不显示，agentbar 为 D/D 且居中。
操作
点击模拟图标4隐藏，再次点击图标4恢复。
画面变化
0ms 图标缩小淡出；100ms 后后续图标补位。恢复时后续图标先让位，目标图标延迟淡入放大。
最终结果
图标4 在自身中心缩放，后续图标按原来的间隔补位/归位。
检查重点
观察缩放中心是否在图形中央，以及补位是否保持原间隔。
02 · 图标列表基础

连续隐藏与中间恢复

前提
初始状态，图标2、图标4 都正在显示。
操作
依次隐藏图标2、图标4，再恢复图标2。
画面变化
每次隐藏都先缩小淡出；100ms 后其后的图标按已隐藏数量向前补位。恢复中间项时，后续项反向让位。
最终结果
图标4 的隐藏状态保持，图标2 恢复后插回原顺序。
检查重点
确认多项消失时位移量按隐藏项数量累计。

添加图标但空间足够

前提

04 · 图标列表自动折叠

添加图标导致尾部图标被收起

前提
初始 D/D agentbar，图标列表与 agentbar 之间只剩有限空间。
操作
点击模拟图标7。
画面变化
图标7 被加入列表，但右侧空间不足；尾部图标被临时收进紫色 more，紫色 more 晚 100ms 出现。
最终结果
紫色 more 出现在不会贴住 agentbar 的位置，被收起的图标隐藏在 more 后面。
检查重点
确认最少留 10px，不让图标列表和 agentbar 贴在一起；more 按 86px 宽度参与计算。
05 · 图标列表自动折叠

more 前移承载多个图标被收起

前提
初始状态，连续添加图标7、图标8。
操作
先点击图标7，再点击图标8。
画面变化
第二个图标加入后，如果 more 放在最后一个位置仍然会贴近 agentbar，它会继续向前寻找安全位置。
最终结果
紫色 more 前移，多枚尾部图标被临时收起。
检查重点
确认 more 位置按 86px 宽度与 10px 安全距离重新计算。
06 · 图标列表自动折叠

隐藏前方图标但仍需 more

列表更紧凑，但空间仍不足，紫色 more 继续存在。
检查重点
确认被收起区第一个图标可以补入，其他被收起状态保持。
07 · 图标列表自动折叠

隐藏前方图标后空间足够

08 · 紫色 more 展开/收起

more 展开与收起

前提
先添加图标7、图标8，形成紫色 more 和多个被收起的图标。
操作
点击紫色 more 展开，再次点击收起。
画面变化
展开：more 移到列表末尾后一位，agentbar 同距右移并淡出；第一个被收起图标 50ms 后出现。收起：被收起图标从后往前隐藏，100ms 后 more 和 agentbar 回归。
最终结果
展开态展示用户已经加入列表的全部图标，收起态恢复当前空间下的折叠布局。
检查重点
确认 agentbar 位移距离等于 more 位移距离。
09 · 紫色 more 展开/收起

展开期间继续添加图标

前提
先添加图标7 并展开紫色 more。
操作
展开态点击模拟图标8。
画面变化
图标8 加入列表后，more 继续向后移动，agentbar 跟随修正偏移，新增图标按展开出现规则淡入。
最终结果
展开态保持，用户已加入列表的图标全部显示。
检查重点
确认展开态不会因 agentbar 安全距离重新折叠。
10 · 紫色 more 展开/收起

展开期间减少到不需要 more

前提
先添加图标7、图标8 并展开紫色 more。
操作
展开态移除图标8，再移除图标7。
画面变化
图标减少后重新评估折叠布局；当不再需要收起任何图标时，more 执行消失动画，agentbar 回中并淡入。
最终结果
退出展开态，紫色 more 隐藏。
检查重点
确认不需要 more 时不会保留一个空的展开入口。

12 · Agentbar 胶囊

当前胶囊隐藏与恢复

操作

14 · 联动与边界

agentbar 变宽导致图标预先被收起

前提
先隐藏左 D 释放空间，再添加图标7，使它在窄 agentbar 下可显示。
操作
再次点击左4恢复 D，使 agentbar 变宽。

agentbar 变宽不会与图标列表交叉。
检查重点
确认图标隐藏早于 agentbar 变宽。

点击左4隐藏左 D，让 agentbar 收缩。

胶囊先消失；50ms 后 agentbar 收缩，图标列表同步补回，紫色 more 淡出。

16 · 联动与边界

展开期间 agentbar 宽度变化后收起

前提
先添加图标7、图标8 并展开紫色 more，agentbar 已淡出并右移。
操作
展开期间点击左1切换胶囊，再点击紫色 more 收起。
画面变化
展开期间 agentbar 内部宽度更新；收起时按最新 agentbar 宽度重新计算 more 回归位置。

收起后 agentbar 以新宽度居中，more 回到当前正确折叠位置。

17 · 联动与边界

动画中再次点击被忽略

前提
初始 D/D agentbar。
操作
快速点击左1后立刻点击左2。
画面变化
左1 触发 D→A 动画；动画还没结束时，左2 的点击会被忽略。
最终结果
最终仍停留在左 A，而不是切到 B。
检查重点
这是当前稳定策略，后续可讨论是否要支持排队或打断。
18 · 联动与边界

极少图标与无 more 状态

前提
初始图标列表有头像 + 图标1-图标6。
操作
连续隐藏图标6、图标5、图标4、图标3。
画面变化
每个图标按 1→0.5、1→0 淡出，后续图标依次补位；由于图标数量减少，不出现紫色 more。
最终结果
列表进入极少图标状态，agentbar 不受影响。
检查重点
确认低数量状态没有空 more 或异常跳位。

建议继续讨论的情况

下面这些不是 bug，而是提案和后续优化时值得明确的产品/动效选择。建议你讲解时把它们作为“已识别的边界”和“可优化方向”。

  • 动画中输入目前是忽略策略。提案里需要明确：这是为了状态稳定，还是后续要改成排队/打断。
  • 图标列表展开后 agentbar 会淡出，用户是否还能理解两者的空间关系；如果需要更强叙事，可以保留一个很淡的位置提示。
  • 紫色 more 展开时继续加入大量图标，列表可能超过 1600 画布宽度；后续可考虑横向滚动、二级收起或限制最大展开数量。
  • 左右胶囊都被隐藏时，目前逻辑会进入接近 AI-only 的状态；这个状态是否真实存在，需要产品侧确认。
  • 图标点击与 agentbar 动画同时发生时，部分输入被锁，部分输入仍可触发；后续可统一输入策略。
  • 透明度也使用弹性动画曲线，可能产生轻微弹性观感；如果提案更偏系统级克制，可以考虑透明度改成更干净的淡入淡出。
  • 150ms 内压缩多个被收起的图标隐藏节奏很紧；若图标很多，建议讨论是否提高窗口或分批隐藏。
  • 模拟层灰底表示用户已经选择了这个图标，不等于它一定正在画面上显示；提案讲解时需要说清楚。
  • 紫色 more 与黄色更多同名但职责不同：紫色代表图标列表还有内容被收起，黄色更多是 agentbar 自己的一部分。
  • 目前案例覆盖代表性路径，没有为 A/B/C/D 所有宽度组合逐个录制；如果提案关注某个胶囊宽度，可追加专项案例。
